# APPLiED

**APPLiED** is a job application assistant built for the intersection of career prep and EdTech.

The name means three things at once:
- **Apply** — helping you apply for jobs
- **Apple** — a nod to education (the teacher's apple)
- **ED** — EdTech, because this tool is built with students and early-career folks in mind

It combines a Flask backend, a LangGraph AI agent, Google Gemini, and a Supabase PostgreSQL database to score job postings against your resume, tailor your application materials, and track where you are in the hiring process.

---

## What it does

- **Sign up / log in** with a real account (passwords hashed with bcrypt via werkzeug)
- **Onboarding** — set your job preferences, target role, location, work setting
- **Profile** — store your resume text and full job search preferences
- **Browse jobs** — view listings fetched from Remotive, filter by title/company/location, save jobs to your tracker
- **Run the agent** — scores every job posting against your resume using TF-IDF + skill matching, re-ranks the top results with Gemini, saves matches to the database
- **Tailor applications** — paste any job description and get a tailored resume + cover letter generated by Gemini
- **Dashboard** — see your application pipeline across all statuses (saved → drafting → ready to review → applied → interviewing → offered → rejected)

## Getting started

### 1. Clone and enter the repo

```bash
git clone <repo-url>
cd APPLiED
```

### 2. Create and activate a virtual environment

```bash
cd backend
python3 -m venv venv
source venv/bin/activate       # macOS / Linux
# or on Windows:
# venv\Scripts\activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Set up your `.env`

Create a `.env` file in the **repo root** (next to `README.md`, not inside `backend/`):

```text
DATABASE_URL=postgresql://user:pass@host:port/dbname
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SECRET_KEY=your-supabase-service-role-key
SUPABASE_BUCKET=application-drafts
GEMINI_API_KEY=your-gemini-api-key
GEMINI_MODEL=gemini-2.0-flash
SECRET_KEY=your-random-secret-key
```

Generate a good `SECRET_KEY` with:
```bash
python3 -c "import secrets; print(secrets.token_hex(32))"
```

### 5. Run the server

```bash
# make sure you're in backend/ with the venv active
python3 app.py
```

The server starts on **http://127.0.0.1:5001**. On first boot it will automatically add a `password_hash` column to your `users` table if it doesn't already exist.

## Testing endpoints

All examples use `curl`. Replace UUIDs and values with real ones from your database.

### Auth

```bash
# sign up
curl -X POST http://localhost:5001/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"email":"ada@example.com","password":"secret123","first_name":"Ada","last_name":"Lovelace"}'

# log in
curl -X POST http://localhost:5001/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"ada@example.com","password":"secret123"}'

# check current session (needs a session cookie from login)
curl http://localhost:5001/auth/me
```

### Health

```bash
curl http://localhost:5001/health
curl http://localhost:5001/health/db
```

### Profile

```bash
# save / update profile
curl -X POST http://localhost:5001/profiles \
  -H "Content-Type: application/json" \
  -d '{
    "email": "ada@example.com",
    "first_name": "Ada",
    "last_name": "Lovelace",
    "resume_text": "Experienced software engineer...",
    "preferences": {
      "job_query": "software engineer",
      "location": "San Francisco",
      "experience_level": "mid",
      "employment_type": ["full_time"],
      "work_setting": ["remote"]
    }
  }'

# fetch profile by user_id
curl http://localhost:5001/profiles/<user_id>
```

### Jobs

```bash
# list jobs (returns up to 100)
curl http://localhost:5001/jobs?limit=100

# ingest new listings from Remotive
curl -X POST http://localhost:5001/jobs/ingest \
  -H "Content-Type: application/json" \
  -d '{"limit": 50}'
```

### Agent (job matching)

```bash
# score all jobs against a user's resume and save matches
curl -X POST http://localhost:5001/agent \
  -H "Content-Type: application/json" \
  -d '{"user_id":"<user_id>"}'
```

### Applications

```bash
# list all applications for a user
curl http://localhost:5001/applications?user_id=<user_id>

# save a job to the tracker (status: saved)
curl -X POST http://localhost:5001/applications/save \
  -H "Content-Type: application/json" \
  -d '{"user_id":"<user_id>","job_posting_id":"<job_id>"}'

# generate a draft resume + cover letter for the top match
curl -X POST http://localhost:5001/applications/draft \
  -H "Content-Type: application/json" \
  -d '{"user_id":"<user_id>","use_top_match":true}'
```

### Tailor

```bash
# tailor resume + cover letter for a custom job description
curl -X POST http://localhost:5001/tailor/generate \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "<user_id>",
    "job_title": "Backend Engineer",
    "company": "Acme Corp",
    "job_description": "We are looking for a Python engineer...",
    "mode": "both"
  }'
# mode options: "both" | "resume" | "cover"
```

## Developing

- **Add a new feature** → create a new file under `backend/services/`, define a `Blueprint`, export it from `services/__init__.py`, and register it in `app.py`.
- **`app.py` stays thin** — only blueprint registration, route-to-template mappings, and the `/agent` POST handler live there.
- **Config lives in `.env`** — access values via `config.py`, never `os.getenv()` directly in service files.
- **Frontend is Jinja2 + vanilla JS** — no build step. Templates extend `base.html`. Auth state is managed via Flask sessions (server-side) with localStorage as a client-side cache.
- **Auth guard** — `base.html` redirects unauthenticated users away from protected routes (`/dashboard`, `/browse`, `/tailor`, `/chat`, `/profile`, `/settings`, `/about`) using a synchronous localStorage check before the page renders, then verifies against `GET /auth/me` in the background.

---

## Notes

- The `password_hash` column is added to `users` automatically at startup — no manual migration needed.
- Gemini is used in two places: the agent's re-ranking step (`graph/graph_builder.py`) and the tailor/draft endpoints (`services/draft_generator.py`, `services/tailor.py`).
- Job listings are pulled from the [Remotive API](https://remotive.com/remote-jobs/api) via `POST /jobs/ingest`.
- Application draft files (tailored resume + cover letter) are saved to Supabase Storage under the configured bucket.
